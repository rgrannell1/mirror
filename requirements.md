The albums.md file currently has a `country` column where each album is tagged with human-readable country names like "Ireland" or "Germany". These get looked up by slug to resolve canonical place URNs, and are emitted as `country` relation triples for albums.

The system should instead use a `places` column in albums.md, where album place values are stored directly as place URNs (e.g. `urn:ró:place:148`) matching the existing format used in photos.md. This removes the need for the slug-to-URN lookup at build time and allows albums to be associated with any place in things.toml, not just those with a country feature.

The DB relation storing album place flags changes from `county` to `places`, and the semantic triple relation emitted for albums changes from `country` to `location`. All existing albums retain their place associations, expressed as place URNs rather than plain text.

The frontend (photos.rgrannell.xyz) is updated to read the `location` relation on albums rather than `country`, and to display place links using the existing place/country link components. The country filter on the albums page continues to work as before, since the place URNs for country-level places are unchanged.

The mirror CLI supports a `fetch` subcommand that automates importing media from a connected camera. The command accepts `--from` and `--to` date arguments expressed as human-readable strings (e.g. "today", "two days ago", "2026-05-07"), an optional `--camera` argument defaulting to `/media/rg/PROGRADE/DCIM`, and reads — never moves — files from the camera DCIM directory. Only files whose creation date (from EXIF if available, falling back to file modification time) falls within the specified inclusive date range are included. Matching photos and videos are copied concurrently to a directory named `~/RawMedia/<start-date>_<end-date>/`. Once copying is complete, the command invokes `badger cluster` on the destination directory to cluster the media by shooting time into numbered subdirectories. When badger finishes, the destination folder is opened in Nautilus. The existing `mirror` pipeline command (no subcommand) is unaffected.

The mirror CLI supports a `copy` subcommand that moves the most recently modified folder under `RAW_MEDIA_DIRECTORY` into the managed media library. It interactively prompts for an album title. It copies that folder to `PHOTO_DIRECTORY/<current-year>/<title>/` and creates an empty `Published/` subdirectory within that destination. After copying, it opens the destination folder in Nautilus.
